# TUI 任务拆分

## 核心目标
构建可用的 TUI 界面，支持 Agent 交互

## 阶段 1: 骨架搭建
目标: 能启动，能看到界面

### 任务 1.1: 最简 App
- [ ] 创建 app.py，继承 textual App
- [ ] 添加 styles.css，设置基本颜色
- [ ] 实现启动和退出
- 验收: `uv run dmcc run` 显示窗口，按 Q 退出

### 任务 1.2: 屏幕切换
- [ ] 创建 HomeScreen（只有 New Session 按钮）
- [ ] 创建 SessionScreen（空白容器）
- [ ] 实现按 'n' 切换
- 验收: 能看到两个屏幕，能切换

## 阶段 2: 输入输出
目标: 能输入，能看到自己的消息

### 任务 2.1: 输入组件
- [ ] InputArea: TextArea + Button
- [ ] Enter 发送，Shift+Enter 换行
- [ ] 发送后清空输入框
- 验收: 输入文字，按 Enter，输入框清空

### 任务 2.2: 消息显示
- [ ] ChatView: 垂直滚动容器
- [ ] UserMessage: 显示用户消息（蓝色背景）
- [ ] 消息追加到 ChatView
- 验收: 输入后能在 ChatView 看到自己的消息

### 任务 2.3: 消息样式
- [ ] UserMessage: 右对齐，蓝色
- [ ] AssistantMessage: 左对齐，绿色
- [ ] 时间戳显示
- 验收: 消息样式清晰可区分

## 阶段 3: Agent 集成
目标: Agent 能响应，结果能显示

### 任务 3.1: 连接 Agent
- [ ] SessionScreen 初始化 Agent
- [ ] 用户输入后调用 Agent.run()
- [ ] 显示 Agent 返回的文本
- 验收: 输入消息，能看到 Agent 回复

### 任务 3.2: 异步处理
- [ ] 使用 asyncio.create_task() 调用 Agent
- [ ] 添加 loading 状态（转圈）
- [ ] 防止重复提交（发送后禁用按钮）
- 验收: Agent 处理时 UI 不卡，有 loading 提示

### 任务 3.3: 工具结果
- [ ] 创建 ToolResultMessage 组件
- [ ] 显示工具调用和结果
- [ ] 支持展开/折叠
- 验收: Agent 使用工具时能看到结果

## 阶段 4: 侧边栏
目标: 信息显示完整

### 任务 4.1: 基础信息

### 任务 4.2: TODO 列表
